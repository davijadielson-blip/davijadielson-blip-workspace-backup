
# Fluxo de Aprovação de Ativos Criativos

Este documento descreve o fluxo padrão para revisão e aprovação de materiais criativos.

## 1. Criação e Rascunho
*   **Responsável:** Criativo (Designer, Redator, Editor)
*   **Ação:** Criação do primeiro rascunho do material (briefing, roteiro, layout, edição inicial).
*   **Ferramenta:** Software de design, edição, editor de texto, ferramenta de gestão de projeto (Notion, Asana).

## 2. Revisão Interna (CQ Criativo)
*   **Responsável:** CCO / Gerente Criativo / Mestre de Pauta
*   **Ação:** Verificação de aderência ao briefing, objetivos, público, mensagem central, requisitos técnicos e de marca. Feedback para ajustes.
*   **Ferramenta:** Ferramenta de gestão de projeto, comentário em arquivos, reunião rápida.
*   **Status:** Em Revisão -> Ajustes Necessários / Aprovado para Cliente

## 3. Revisão do Cliente
*   **Responsável:** Cliente / Stakeholder principal
*   **Ação:** Verificação geral do material, alinhamento estratégico e de negócio. Feedback para ajustes.
*   **Ferramenta:** E-mail, ferramenta de gestão de projeto, reunião específica.
*   **Retorno:** Aprovado / Ajustes Necessários (retorna ao passo 1 ou 2, dependendo da complexidade)

## 4. Ajustes Finais
*   **Responsável:** Criativo / Editor
*   **Ação:** Implementação dos ajustes solicitados pelo cliente/revisão interna.
*   **Ferramenta:** Software de design, edição, etc.

## 5. Aprovação Final
*   **Responsável:** Cliente / Stakeholder principal (ou quem tiver o poder de decisão final)
*   **Ação:** Confirmação de que o material está pronto para ser publicado ou utilizado.
*   **Ferramenta:** E-mail formal, status em ferramenta de gestão, confirmação via chat.
*   **Status:** Aprovado / Reprovado (raro, volta para fase de ajuste)

## 6. Entrega / Publicação
*   **Responsável:** Equipe de Produção / Social Media / Áreas responsáveis
*   **Ação:** Upload e publicação final do material aprovado na plataforma definida.
*   **Ferramenta:** Plataforma de publicação (Social Media Tools, CMS, etc.)

---
**Observações:**
*   Prazos devem ser definidos para cada etapa.
*   Em caso de urgência, o fluxo pode ser simplificado com aprovação direta do CCO ou do cliente em etapas específicas.
*   Todos os feedbacks e aprovações devem ser registrados.
