# Aprendizado aprovado — Status no Notion

## Regra definida por Jadielson
Para conteúdos/rotas/calendários lançados no Notion na database `Produção & Agenda — LÓGIKA`, o campo **Status** deve ficar sempre como:

> **A iniciar**

## Aplicação
- Não usar “Aguardando aprovação” como status padrão de criação.
- A validação/observação de que depende de Jadielson/SMS deve ficar em **Observações** ou no corpo da página, não no campo Status.
- Para futuras importações no Notion, Jarvis deve criar os itens já com `Status = A iniciar`.

## Observação
Esta regra vale especialmente para as frentes de produção da Saúde/Social Media.
